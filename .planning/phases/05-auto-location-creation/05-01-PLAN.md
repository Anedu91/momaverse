  <action>
Create `pipeline/location_resolver.py` with a single public function:

```python
def resolve_locations(raw_json_data, website_id, cursor, connection):
```

**Parameters:**
- `raw_json_data`: The parsed JSON dict (pre-date-filter, navigated to data_path level -- the events dict keyed by event ID)
- `website_id`: The website ID (for scoping alternate names)
- `cursor`, `connection`: DB cursor and connection

**Returns:** int -- count of locations created

**Implementation steps inside resolve_locations():**

1. **Extract unique venues** from raw_json_data:
   - Navigate: `event > lugares > lugar_id > {nombre, direccion, zona, lat, lng}`
   - Build a dict keyed by lugar_id to deduplicate across events
   - Skip venues with missing/zero lat or lng (parse as float, skip if 0.0 or None)
   - Skip venues with empty nombre

2. **Buenos Aires bounds check** -- define constants at module level:
   ```python
   BA_BOUNDS = {
       'lat_min': -34.75, 'lat_max': -34.50,
       'lng_min': -58.60, 'lng_max': -58.28
   }
   ```
   Skip any venue where lat/lng falls outside these bounds.

3. **Name normalization:**
   - Import `normalize_event_name_caps` from processor.py for title-casing UPPERCASE venue names
   - Import `_normalize_location_name` from processor.py for dedup matching
   - Apply `normalize_event_name_caps(nombre)` to get display name
   - Apply `_normalize_location_name(display_name)` to get matching key

4. **Load existing locations** for dedup:
   - Call `db.get_all_locations(cursor)` to get all current locations
   - Build a set of normalized names (using `_normalize_location_name` on each location name and all alternate names)
   - Build a list of (lat, lng) tuples for coordinate proximity checking

5. **Haversine distance function** (module-level helper):
   ```python
   import math

   def _haversine_meters(lat1, lng1, lat2, lng2):
       R = 6371000
       phi1, phi2 = math.radians(lat1), math.radians(lat2)
       dphi = math.radians(lat2 - lat1)
       dlambda = math.radians(lng2 - lng1)
       a = math.sin(dphi/2)**2 + math.cos(phi1)*math.cos(phi2)*math.sin(dlambda/2)**2
       return R * 2 * math.atan2(math.sqrt(a), math.sqrt(1-a))

   PROXIMITY_THRESHOLD_METERS = 100
   ```

6. **Dedup cascade** for each venue (in order):
   a. **Normalized name match**: If `_normalize_location_name(display_name)` exists in the set of existing normalized names, skip (already exists)
   b. **Coordinate proximity**: If any existing location is within 100m (haversine), skip
   c. If neither match, this is a new venue -- insert it

7. **Insert new location:**
   ```sql
   INSERT INTO locations (name, address, lat, lng, emoji)
   VALUES (%s, %s, %s, %s, %s)
   ```
   - `name`: title-cased display name from step 3
   - `address`: `direccion` + `, ` + `zona` if zona exists (e.g., "Pasaje Santos Discepolo 1857, Abasto")
   - `lat`, `lng`: float values from API
   - `emoji`: `'🎭'` (theater emoji -- these are theater sources)
   - After insert, get `cursor.lastrowid` for the new location ID

8. **Insert website-scoped alternate name:**
   ```sql
   INSERT INTO location_alternate_names (location_id, alternate_name, website_id)
   VALUES (%s, %s, %s)
   ```
   - `alternate_name`: the ORIGINAL nombre from the API (uppercase, as-is) -- this is how get_location_id() will match events from this website
   - `website_id`: the website_id parameter
   - This ensures the processor's website-scoped lookup finds the venue

9. **Add to existing sets** after insert (so subsequent venues in same batch don't duplicate):
   - Add normalized name to the names set
   - Add (lat, lng) to the coordinates list

10. **Commit and return** count of created locations. Call `connection.commit()` once after all inserts.

**Print logging** (match pipeline style):
- `"  Resolving locations from {len(unique_venues)} unique venues..."`
- `"    - Skipped {name}: outside BA bounds"` (only if verbose/debug, or just count)
- `"    - Created {count} new location(s), skipped {skip_count} existing"`
