<objective>
Add JSON API crawling to the pipeline so sites with structured endpoints (starting with Alternativa Teatral) are crawled via lightweight HTTP GET instead of browser rendering.

Purpose: Alternativa Teatral exposes a JSONP endpoint with ~920 events. Browser crawling fails on its infinite-scroll JS. HTTP GET + JSON parse is faster, cheaper, and more reliable. The JSON-to-markdown step replaces only the browser -- Gemini still handles extraction.

Output: Working json_api crawl path that produces markdown identical in purpose to browser-crawled content, feeding into the existing extraction/processing/merge pipeline unchanged.
</objective>

<task type="auto">
  <name>Task 2: JSON API crawl function (crawler.py)</name>
  <files>pipeline/crawler.py</files>
  <action>
Add these functions to `crawler.py`, after the existing imports and before `resolve_url_templates()`:

1. Add imports at the top (after existing imports):
   ```python
   import json
   import re
   import httpx
   ```

2. Add helper functions:

   **strip_jsonp(text, callback_name=None):** Strip JSONP callback wrapper to get pure JSON string. If callback_name provided, strip that exact prefix. Otherwise use generic regex `^[a-zA-Z_]\w*\s*\((.*)\)\s*;?\s*$` with re.DOTALL. Return text unchanged if neither matches (already plain JSON).

   **filter_by_date_window(events_dict, days_ahead=30):** Filter events dict (keyed by event ID) to only those with at least one upcoming `proxima_fecha` within the window. Navigate `event > lugares > * > funciones > * > proxima_fecha`. Parse dates with `datetime.strptime(proxima, '%Y-%m-%d %H:%M')`. Include events with NO proxima_fecha at all (they may still be relevant -- let Gemini decide). Only exclude events whose ALL dates are in the past or beyond the window.

   **flatten_events_to_markdown(events_dict, fields_include=None):** Convert filtered JSON events to markdown. For each event:
   - `## {titulo}`
   - `**Tags:** {comma-joined clasificaciones.*.descripcion}`
   - For each lugar: `**Venue:** {nombre}`, `**Address:** {direccion}, {zona}`
   - For each funcion: `- {dia} {hora} (next: {proxima_fecha})`
   - `**URL:** https://www.alternativateatral.com/{url_slug}` (if url exists)
   - `**Tickets:** {url_entradas}` (if exists)
   - Blank line between events
   - If fields_include is provided, only include those top-level keys when iterating (optimization for future sources -- for now Alternativa Teatral uses all listed fields).

3. Add main function:

   **async def crawl_json_api(website, cursor, connection, crawl_run_id):**
   - Get `config = website.get('json_api_config', {})`; if empty, print skip message and return None
   - Create safe_filename and crawl_result_id via existing db functions (same pattern as crawl_website)
   - HTTP GET using `httpx.AsyncClient(timeout=30)` to `config.get('base_url')` (fall back to `website['urls'][0]['url']` if no base_url)
   - Strip JSONP if `config.get('jsonp_callback')` is set
   - `json.loads()` the result
   - Navigate data_path: `for key in config.get('data_path', '').split('.'):` if data_path is truthy
   - Filter by date window: `config.get('date_window_days', 30)`
   - Flatten to markdown with `config.get('fields_include')`
   - If markdown is empty after filtering, call `db.update_crawl_result_failed()` with "No events after filtering" and return None
   - Otherwise call `db.update_crawl_result_crawled()` with the markdown, update last_crawled, print stats, return crawl_result_id
   - Wrap everything in try/except, on error call `db.update_crawl_result_failed()` and return None
   - Always call `db.update_website_last_crawled()` (same pattern as crawl_website)

   Use `response.text` (not `.content.decode()`) to let httpx handle encoding.
